# Workflow: Revisione periodica della knowledge base

Usa questo workflow per controllare qualita, coerenza e manutenzione della knowledge base.

## Passaggi

1. Fornire alla chat l'elenco dei file.
2. Fornire `.llm/system-context.md`.
3. Usare `.llm/prompts/summarize-repository.md`.
4. Identificare file obsoleti.
5. Identificare duplicazioni possibili.
6. Raccogliere prossime azioni sparse.
7. Proporre eventuali archiviazioni.
8. Non modificare nulla senza conferma.

## Output atteso

- Progetti attivi.
- Aree principali.
- Idee aperte.
- Decisioni recenti.
- Prossime azioni aggregate.
- File obsoleti o duplicati.
- Suggerimenti di manutenzione.

## Regole

- Non inventare informazioni.
- Non fare modifiche automatiche.
- Usare `da_verificare` per dubbi o incoerenze.
